# 0021. Upstream-first Chocolatey-for-Wine bootstrap

Status: accepted
Date: 2026-07-10
Owner: Noah Giroux / CTO
Supersedes: [0020. Deterministic Chocolatey-for-wine MVP reconstruction](0020-deterministic-chocolatey-for-wine-mvp.md)

## Decision

Cage's `chocolatey` module will use the pinned Chocolatey-for-Wine release installer (`ChoCinstaller_0.5c.755.exe /s /q`) as the compatibility bootstrap mechanism rather than manually reconstructing its native CLR, registry, PowerShell-wrapper, and app-local loader state.

Cage retains its independent success boundary:

1. verify and extract the pinned Chocolatey-for-Wine release;
2. run the upstream installer with a bounded timeout and a module-cache-backed `CFW_CACHE`;
3. require canonical `C:/ProgramData/chocolatey/bin/choco.exe` after the installer settles;
4. run bounded `choco --version` and `choco source list` readiness probes;
5. apply feature policy and prove a local package install/uninstall lifecycle before user-requested packages.

The upstream installer exit code is recorded as evidence, not accepted as success by itself.

## Reasoning

The manual MVP reconstruction repeatedly stalled at the managed `choco.exe` startup boundary despite correctly materializing the expected CLR files and registry entries. It was an incomplete recreation of upstream compatibility behavior. The selected path follows the working upstream integration as a complete compatibility environment while preserving Cage-owned canonical file, command-readiness, lifecycle, timeout, and diagnostic gates.

## Consequences

- The active profile is `cfw-v0.5c.755-choco-2.6.0-upstream-r6`.
- Manual .NET payload/registry reconstruction and app-local CLR closure are no longer generated by the Chocolatey module.
- The upstream installer may still fail or exit without creating canonical Chocolatey; Cage fails closed in that case and uploads diagnostic evidence.
- Revisit this decision if the upstream installer consistently fails in the Cage Wine runtime, the upstream release layout changes, or a reproducible end-to-end package lifecycle passes and a pre-baked image becomes warranted.
